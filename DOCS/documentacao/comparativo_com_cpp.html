Diferenças cruciais entre C# e C++

Este módulo tende a ser útil a programadores C++ que querem programar em C#.

Página do projeto: http://tux.lcc.ufrn.br/~iuri/projeto 

Exemplos usados no módulo:

Exemplo 1, imprimindo um inteiro:

feito para C++:


void main() {
    int x = 16;
    std::cout << "Oi vocês, olhe este numero: "<< x << endl;
}
 

convertido para C#:


class Programa1 {
    public static void Main(){
        int x = 16;
        System.Console.WriteLine("Oi vocês, olhe este número: {0}", x );
    }
}
 

Exemplo 2, chamando uma função/método 1:

feito em C++:


using std::cout;
void produto(int, int);

int main() {
    produto(16, 3);
    return(0);
}

void produto(int x, int y) {
   cout << x * y << endl; 
}
 

Convertido para C#: 


using System;
class Programa2 {
    public static int Main() {
        Produto(16, 3);
        return(0);
    }

    static void Produto(int x, int y) {
       Console.WriteLine(x * y); 
    }
}
 

Exemplo 3, chamando uma função/método 2:

feito em C++:


using std::cout;
int soma(int, int);

int resultado; //variável global

void main() {
    int x = 16;
    int y = 20;

    cout << soma(x,y) << endl;
    soma(45, 88);
    cout << resultado << endl;
}

int soma(int num1, int num2) {
    resultado = num1 + num2;
    return(resultado);
}
  

convertido para C#:


using System;

class Programa3 {

    static int resultadoDaSoma; //propriedade global

    public static void Main() {
        int x = 16;
        int y = 20;

        //Chamando um método explicitando a classe
        Console.WriteLine( Programa3.Soma(x,y) );

        //Chamando o mesmo método sem explicitar a classe
        Soma(45, 88); 

        //mostrando o valor sem explicitar o nome da classe
        Console.WriteLine(resultadoDaSoma); 

    }

    static int Soma(int num1, int num2) {
        resultadoDaSoma = num1 + num2;
        return(resultadoDaSoma);
    }
}
 

Exemplo 4, usando o switch()

escrito em C++:


using std::cout;
using std::cin;
void main() {

    int numero;
    cout << "Digite um número de 0 a 3: ";
    cin >> numero;
 
   switch(numero) {
        case 3:
            cout << "igual a 3" <<endl;
        case 2:
            cout << "maior ou igual a 2" <<endl;
            break;
        case 1:
            cout << "igual a 1" <<endl;
        case 0:
            cout << "maior ou igual a 0" <<endl;
            break;
    }
}
 

convertido para C#:


using System;
class Programa4{
    public static void Main() {

        int numero;
        Console.Write("Digite um número de 0 a 3: ");
        numero = int.Parse( Console.ReadLine() );
 
       switch(numero) {
            case 3:
                Console.WriteLine( "igual a 3" );
                goto case 2;
            case 2:
                Console.WriteLine( "maior ou igual a 2" );
                break;
            case 1:
                Console.WriteLine( "igual a 1" );
                goto case 0;
            case 0:
                Console.WriteLine( "maior ou igual a 0" );
                break;
        }
    }
}

Exemplo 5, imprimir quadrados

escrito em C++:


using std::cin;
using std::cout;

void main() {

    int x;

    cout << "Digite um número, 0 para sair: ";
    cin >> x;

    while (x) {

        cout << x * x << endl;

        cout << "Digite um número, 0 para sair: ";
        cin >> x;     
    }

}
 

convertido para C#:


using System;
class Programa5 {
    public static void Main() {
        
        int x;

        Console.Write ( "Digite um número, 0 para sair: ");
        x = int.Parse( Console.ReadLine() );
        
        while (x != 0 ) {

            Console.WriteLine ( x * x );

            Console.Write ( "Digite um número, 0 para sair: ") ;
            x = int.Parse( Console.ReadLine() );

        }

    }

}
  

Classe obrigatória

Todo programa ou biblioteca em C#, tem que ter pelo menos uma classe, entenda isto como se todas as suas ferramentas de C++ devem, em C#, obrigatoriamente estar dentro de uma estrutura de dados (classe).

Observando o programas convertidos, podemos ver que as antigas funções em C++  e suas antigas variáveis globais  estão agora dentro de uma classe.

Não existem mais funções e variáveis globais.

De variáveis globais para propriedades.

Para declarar suas antigas variáveis globais, basta declará-las fora do método Main(), mas destro da classe  obrigatória, como propriedades da classe(variáveis estáticas), você pode evidenciar isto no exemplo 3.

De funções para métodos estáticos. 

Em C# não existe funções, pelo menos não ao pé letra, basta declará-las normalmente como sendo métodos estáticos da classe obrigatória. Observe que no exemplo 3 podemos chamar as nossas antigas funções especificando a classe(como métodos estáticos) e sem especificar a classe (como funções).

Observe que chamamos várias vezes o método WriteLine da classe Console.

A palavra reservada static

Quando seus dados estão dentro de uma classe, eles então só podem ser usados em objetos, ao declararmos alguma propriedade ou método como static, ela pode ser usada sem ser preciso instanciar um objeto da classe, se não tivéssemos declarado nossas antigas funções como static elas só poderiam ser usadas se antes nós criássemos um objeto da classe obrigatória e chamássemos tudo a partir dele.

Se não declarássemos nossos métodos como estáticos, ao invés de chamar simplesmente a função como chamamos, teríamos que, primeiro, instanciar um objeto da classe obrigatória e então chamar o método

Exemplo de como iríamos chamar um método ele não fosse estático:

se fosse estático: 

Método(); // sem especificar a classe, 
          // caso se esteja chamando de outro método da classe
Classe.Método(); // especificando a classe.

se não fosse estático:

//Método(); // Chamada ilegal.
//Classe.Método(); // Chamada ilegal.
Classe objeto = new Classe(); // criá-se primeiro o objeto.
objeto.Método(); // depois se chama o método a partir do objeto.

Protótipos

Em C#, você não precisa mais prototipar suas funções antes de usá-las, isto na prática quer dizer que não é mais preciso usar arquivos head para que suas classes funcionem corretamente. C# não contém a diretiva de pré-processador #include para importar partes do programa.

Rótulos

Não existem mais rótulos para seus métodos/propriedades, agora você diz o que cada estrutura é, especificando o mesmo com o tipo de visibilidade antes da declaração, caso você deixe de fazer, automaticamente ela será private.

O método Main()

Diferente de C++, o ponto de entrada de um programa se chama Main, com M maiúsculo, ele deve obrigatoriamente ser public porque deve está acessível fora da classe e deve ser static porque deve estar acessível sem ser preciso criar um objeto.

O operador de escopo

Programas escritos em C#, não estão sobrecarregados de operadores "::" ou "->". O ":" não é um operador em C#, o operador "->" é usado somente uma pequena porção dos programas escritos em C#. 

Se você fosse chamar um método fora num contexto de classe(isto é, sem criar um objeto), em C++, você faria isto usando Classe::Método, em C#, você utiliza o operador . como se fosse chamar um método de um objeto ou uma classe como em Console.WriteLine();

Detalhe em C++ para .NET, você chama Console::WriteLine().

O switch

Todo rótulo de um switch agora deve conter um break ou um goto, não é mais possível para ele executar todos os casos abaixo dele até achar um break. Dê uma olhada no exemplo 4.

Conversão de int para bool

Agora você não pode mais converter inteiros para valores lógicos, isto na prática quer dizer que a construção:

if (x) {
    //FacaAlgumaCoisa()
}

não é mais permitida. Dê uma olhada no exemplo 5.
